Exam Overview
Oracle Cloud Infrastructure 2025 Security Professional certification is a Hands-on Performance based exam comprising of hands-on challenges and multiple choice questions.

The Oracle Cloud Infrastructure Security certification is intended for professionals who are responsible for security in their Oracle Cloud Infrastructure environments. The target candidate should have 2+ years of experience in designing and implementing security solutions. Additionally, the candidate should have 6 months or more of hands-on experience in securing workloads on OCI. The exam validates the following OCI Security topics:

• OCI Security Services: Working knowledge of OCI security services to establish a secure cloud environment.

• Identity and Access Management (IAM): Understanding of OCI IAM features, including Identity Domains, Users, Groups, Policies, Compartments, Dynamic Groups, and Advanced Policies for secure access control.

• Network Security: Skills in securing OCI networks through Security Lists, Network Security Groups, Network Firewalls, Web Application Firewalls, certificate lifecycle management, and Load Balancers.

• Workload Protection: Expertise in managing OS and workload security, including the use of OCI Bastion, vulnerability scanning for hosts and containers, and automating updates with OS management.

• Data Protection: Managing encryption keys, secrets, and data protection with OCI Vault and OCI Data Safe.

• Security Operations: Proficiency in maintaining a strong security posture with tools like Cloud Guard, Security Zones, OCI Monitoring, Logging, Events, and alerting systems.

• Compliance: Familiarity with OCI’s compliance frameworks to ensure security standards are met.

Up-to-date OCI training and hands-on experience are highly recommended to ensure readiness for the exam.

In addition to Standard Multiple-Choice Questions, the Hands-on Performance-based exam includes questions that are tied to specific configurations in the given Oracle Cloud Infrastructure environment.

These questions require candidates to review a configuration → assess that configuration → answer the question related to the configuration. Candidates are evaluated on their ability to analyse how various parameters and configurations affect system behaviour and determine which actions or settings result in specific outcomes.
